    Métodos en Java

    Un método es un bloque de código reutilizable que realiza una tarea concreta. Permiten organizar el programa en partes más pequeñas y evitar repetir código.

    ¿Por qué usar métodos?

    Sin métodos vs con métodos

    Sin métodos, si necesitas calcular el área de un rectángulo en 5 partes del programa, escribes la fórmula 5 veces. Con un método, la escribes una vez y la llamas cuando la necesites.

    Estructura de un método

    modificador tipoDeDato nombreMetodo(parámetros) {
    // código
    return valor; // solo si no es void
}
    Parte Descripción Ejemplo
    modificador Visibilidad y comportamiento public static
    tipoDeDato Tipo del valor que devuelve (void si no devuelve) int, String, void
    nombreMetodo Nombre en camelCase calcularArea
    parámetros Datos de entrada (opcional) int base, int alto
    return Devuelve el resultado (obligatorio si no es void) return base * alto;

    Tipos de métodos

    public static void saludar() {
    System.out.println("¡Hola!");
}

// Llamada:
saludar(); // ¡Hola!

    public static void mostrarNombre(String nombre) {
    System.out.println("Nombre: " + nombre);
}

// Llamada:
mostrarNombre("Ana"); // Nombre: Ana

    public static int sumar(int a, int b) {
    return a + b;
}

// Llamada:
int resultado = sumar(5, 3);
System.out.println(resultado); // 8

    public static double calcularArea(double base, double altura) {
    return base * altura;
}

// Llamada:
double area = calcularArea(4.5, 3.0);
System.out.println("Área: " + area); // Área: 13.5

    Palabra clave return

    return hace dos cosas: 1. Devuelve un valor al lugar donde se llamó el método 2. Termina la ejecución del método inmediatamente

    public static String calificar(int nota) {
    if (nota >= 90) return "Sobresaliente";
    if (nota >= 70) return "Notable";
    if (nota >= 50) return "Aprobado";
    return "Suspenso"; // se llega aquí si ninguna condición anterior se cumplió
}

    void no usa return

    Los métodos void no devuelven ningún valor. Puedes usar return; sin valor para salir antes del método, pero no es obligatorio.

    Sobrecarga de métodos

    Java permite tener varios métodos con el mismo nombre siempre que tengan distintos parámetros. A esto se llama sobrecarga:

    public static int sumar(int a, int b) {
    return a + b;
}

public static double sumar(double a, double b) {
    return a + b;
}

public static int sumar(int a, int b, int c) {
    return a + b + c;
}

// Java elige el correcto según los argumentos:
System.out.println(sumar(2, 3));         // 5
System.out.println(sumar(1.5, 2.5));     // 4.0
System.out.println(sumar(1, 2, 3));      // 6

    Métodos en clases (instancia)

    Los métodos que hemos visto son static (de clase). También existen métodos de instancia, que pertenecen a un objeto:

    Calculadora.java
    class Calculadora {
    private int resultado = 0;

    public void sumar(int n) {
        resultado += n;
    }

    public void restar(int n) {
        resultado -= n;
    }

    public int getResultado() {
        return resultado;
    }
}
    Main.java
    public class Main {
    public static void main(String[] args) {
        Calculadora calc = new Calculadora();

        calc.sumar(10);
        calc.sumar(5);
        calc.restar(3);

        System.out.println("Resultado: " + calc.getResultado()); // 12
    }
}

    Métodos con arrays

    Los métodos pueden recibir y devolver arrays:

    public static int[] generarSerie(int n) {
    int[] serie = new int[n];
    for (int i = 0; i < n; i++) {
        serie[i] = i + 1;
    }
    return serie;
}

public static void imprimirArray(int[] array) {
    for (int valor : array) { // for-each
        System.out.print(valor + " ");
    }
    System.out.println();
}

public static void main(String[] args) {
    int[] numeros = generarSerie(5);
    imprimirArray(numeros); // 1 2 3 4 5
}

    Ejemplo completo

    Main.java
    public class Main {

    public static double areaCirculo(double radio) {
        return Math.PI * radio * radio;
    }

    public static boolean esPar(int numero) {
        return numero % 2 == 0;
    }

    public static String repetir(String texto, int veces) {
        String resultado = "";
        for (int i = 0; i < veces; i++) {
            resultado += texto;
        }
        return resultado;
    }

    public static void main(String[] args) {
        System.out.println("Área: " + areaCirculo(5));      // 78.53...
        System.out.println("¿Es par 4? " + esPar(4));       // true
        System.out.println("¿Es par 7? " + esPar(7));       // false
        System.out.println(repetir("Java ", 3));             // Java Java Java
    }
}

    Buenas prácticas

    Diseño de métodos

    • Un método debe hacer una sola cosa bien hecha
    • Nombres descriptivos en camelCase: calcularPrecioConIva mejor que calc
    • Si un método tiene más de 20 líneas, considera dividirlo
    • Evita pasar más de 3-4 parámetros (señal de que hace demasiado)

    Conclusión

    Resumen

    • Los métodos son bloques reutilizables que realizan una tarea concreta
    • Pueden tener parámetros (datos de entrada) y un retorno (valor de salida)
    • void indica que el método no devuelve ningún valor
    • La sobrecarga permite múltiples métodos con el mismo nombre pero distintos parámetros
    • Los métodos de instancia pertenecen a objetos y pueden acceder a sus atributos
