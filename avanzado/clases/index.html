    Clases y Objetos en Java

    Java es un lenguaje orientado a objetos (POO). La programación orientada a objetos organiza el código en torno a clases y objetos, que modelan entidades del mundo real.

    ¿Qué es una clase?

    Una clase es una plantilla o molde que define las propiedades y comportamientos de un tipo de objeto.

    Analogía

    Una clase es como el plano de un edificio: define cómo será el edificio, pero no es el edificio en sí. El edificio construido a partir del plano es el objeto.

    ¿Qué es un objeto?

    Un objeto es una instancia de una clase. Cada objeto tiene sus propios valores para los atributos definidos en la clase.

    // Clase = plantilla
class Persona {
    String nombre;
    int edad;
}

// Objeto = instancia de la clase
Persona p1 = new Persona();
Persona p2 = new Persona(); // otro objeto independiente

    Atributos y métodos

    Una clase puede tener:

    • Atributos — variables que describen las propiedades del objeto
    • Métodos — funciones que definen el comportamiento del objeto
    Persona.java
    class Persona {
    // Atributos
    String nombre;
    int edad;

    // Método
    void saludar() {
        System.out.println("Hola, soy " + nombre + " y tengo " + edad + " años.");
    }
}
    Main.java
    public class Main {
    public static void main(String[] args) {

        Persona p = new Persona();
        p.nombre = "Ana";
        p.edad = 25;

        p.saludar(); // Hola, soy Ana y tengo 25 años.
    }
}

    Constructor

    El constructor es un método especial que se ejecuta automáticamente al crear un objeto con new. Sirve para inicializar los atributos.

    Con constructor
    class Persona {
    String nombre;
    int edad;

    // Constructor
    Persona(String nombre, int edad) {
        this.nombre = nombre; // (1)
        this.edad = edad;
    }

    void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
}

public class Main {
    public static void main(String[] args) {
        Persona p = new Persona("Carlos", 30); // llama al constructor
        p.saludar(); // Hola, soy Carlos
    }
}
    1. this.nombre se refiere al atributo de la clase, para distinguirlo del parámetro con el mismo nombre.

    Constructor por defecto

    Si no defines ningún constructor, Java crea uno vacío automáticamente. En cuanto defines uno propio, el automático desaparece.

    Encapsulación

    La encapsulación consiste en proteger los atributos de una clase declarándolos como private y accediendo a ellos solo mediante métodos públicos llamados getters y setters.

    Clase con encapsulación
    class CuentaBancaria {
    private double saldo; // no accesible desde fuera

    // Constructor
    CuentaBancaria(double saldoInicial) {
        this.saldo = saldoInicial;
    }

    // Getter: obtener el saldo
    public double getSaldo() {
        return saldo;
    }

    // Setter con validación
    public void depositar(double cantidad) {
        if (cantidad > 0) {
            saldo += cantidad;
        }
    }

    public void retirar(double cantidad) {
        if (cantidad > 0 && cantidad <= saldo) {
            saldo -= cantidad;
        } else {
            System.out.println("Saldo insuficiente");
        }
    }
}
    Main.java
    public class Main {
    public static void main(String[] args) {
        CuentaBancaria cuenta = new CuentaBancaria(1000.0);

        cuenta.depositar(500.0);
        System.out.println("Saldo: " + cuenta.getSaldo()); // 1500.0

        cuenta.retirar(200.0);
        System.out.println("Saldo: " + cuenta.getSaldo()); // 1300.0

        // cuenta.saldo = -999; // ❌ Error: saldo es private
    }
}

    ¿Por qué encapsular?

    • Evita que datos internos se modifiquen de forma incorrecta
    • Permite añadir validaciones en los setters
    • Facilita cambiar la implementación interna sin afectar el resto del código

    Modificadores de acceso

    Modificador Acceso desde
    public Cualquier clase
    private Solo dentro de la misma clase
    protected Misma clase y subclases
    (sin nada) Solo dentro del mismo paquete

    Múltiples objetos

    Cada objeto es independiente. Modificar uno no afecta a los demás:

    Persona p1 = new Persona("Ana", 25);
Persona p2 = new Persona("Luis", 30);

p1.saludar(); // Hola, soy Ana
p2.saludar(); // Hola, soy Luis

    Ejemplo completo

    Main.java
    class Producto {
    private String nombre;
    private double precio;

    Producto(String nombre, double precio) {
        this.nombre = nombre;
        this.precio = precio;
    }

    public String getNombre() { return nombre; }
    public double getPrecio() { return precio; }

    public void mostrarInfo() {
        System.out.println(nombre + " → " + precio + "€");
    }
}

public class Main {
    public static void main(String[] args) {
        Producto p1 = new Producto("Teclado", 45.99);
        Producto p2 = new Producto("Ratón", 29.50);

        p1.mostrarInfo(); // Teclado → 45.99€
        p2.mostrarInfo(); // Ratón → 29.5€
    }
}

    Conclusión

    Resumen

    • Una clase es una plantilla; un objeto es una instancia de esa plantilla
    • Los atributos definen las propiedades y los métodos el comportamiento
    • El constructor inicializa los objetos al crearlos con new
    • La encapsulación protege los datos con private y los expone con getters/setters
